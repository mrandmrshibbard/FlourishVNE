‚ö†Ô∏è Common Mistake: Many users struggle with characters because they skip steps or do them in the wrong order. Follow this exact workflow!

Character Creation Workflow

1
Create the Character
Go to Resources ‚Üí Characters tab and click + Add Character. Give it a name (e.g., "Emma").
2
Add Expressions
Click your character, then in the Character Editor tab (right side), click + Add Expression. Create expressions like "happy", "sad", "angry", "neutral".
3
Create Layers
Still in Character Editor, click + Add Layer. Common layers: "body", "face", "outfit", "accessory". Think of layers like Photoshop layers that stack on top of each other.
4
Upload Sprites
For each layer, click üì§ Upload and select your PNG images. Upload ALL the sprite variations you'll need (e.g., happy_face.png, sad_face.png).
5
Assign Sprites to Expressions
Go to Character Inspector tab. For each expression, click ‚úèÔ∏è Edit and select which sprite shows for each layer. Now "happy" can use happy_face.png, "sad" uses sad_face.png, etc.
üí° Pro Tip: You can reuse sprites across expressions! If the body doesn't change, all expressions can use the same body sprite.
‚úÖ You're Done! Your character now has multiple expressions. When you use the Show Character command in scenes, you can choose which expression to display!

Character Layer Tips

  • Z-Index matters: Layers are drawn from lowest to highest number. Body should be z-index 0, face should be 1, accessories 2, etc.
  • Transparent PNGs: Make sure your sprites have transparent backgrounds so they layer properly
  • Consistent size: All sprites for a character should be the same dimensions (e.g., 1000x1500px)
  • Name clearly: Use descriptive names like "emma_body_casual", "emma_face_happy"

üé¨ Creating Scenes

Step 1: Create a Scene

Go to Resources ‚Üí Scenes and click + Add Scene. Name it (e.g., "Chapter 1 - Morning").

Step 2: Add Commands

Click your scene to open the Scene Editor. Click the + button to add commands. Commands execute top to bottom.

Common Commands:

  • Change Background: Set the scene (bedroom, school, park)
  • Show Character: Display a character with a specific expression at a position
  • Show Text: Display dialogue. Use {variableName} to insert variable values
  • Show Choices: Give player branching options
  • Label: Addes a label to a scene
  • Jump or Jump to Label: Move to a specific scene or label
  • Play Music/SFX: Add audio atmosphere
  • Set Variable: Track player choices, stats, relationship points, etc.
  • Branch Start/End: Conditional story paths based on variables

Step 3: Configure Each Command

Click a command in the Scene Editor, then edit its properties in the Properties Inspector panel on the right.

üí° Text Variables: Use {playerName} in dialogue to insert dynamic text. Create variables in Resources ‚Üí Variables first!

üé® Variables (Power User Feature)

Variables let you create dynamic, personalized stories that react to player choices.

Creating Variables

Go to Resources ‚Üí Variables and click + Add Variable.

  • String: Text values (player name, location)
  • Number: Stats, points, counters (affection, money, health)
  • Boolean: True/false flags (has_key, met_emma, game_over)

Using Variables

  • Set Variable: Use the command to change values (set, add, subtract, random)
  • In Text: Type {variableName} in dialogue to display the value
  • Branching: Use Branch Start command with conditions to create different story paths
‚ú® New Feature! Variables can now be set to random ranges! Use the "Random (Range)" operator to generate random numbers between min and max values. Perfect for dice rolls, loot drops, or randomized events!

üé® UI Screens & Menus

Create custom title screens, pause menus, settings screens, and more!

Creating a UI Screen

  1. Go to Resources ‚Üí UI Screens and click + Add Screen
  2. Name it (e.g., "Title Screen", "Pause Menu")
  3. Click the screen to open the Menu Editor
  4. Add elements: Buttons, Text, Images, Videos, Save Slots, Settings
  5. Position and style each element using the UI Element Inspector

UI Element Types

  • Button: Clickable actions (Start Game, Save, Load, Go to Screen)
  • Text: Labels and headings
  • Image: Logos, decorations, backgrounds
  • Video: Animated backgrounds or cutscenes
  • Save Slot Grid: Automatic save/load interface
  • Settings Slider: Volume controls (music, sfx, text speed)
  • Settings Toggle: On/off options (auto-advance, skip read text)
üí° Special Screens: Set your Title Screen and Pause Screen in Resources ‚Üí UI Screens at the top. These screens appear automatically at game start and when pressing ESC.

üéÆ Testing Your Game

Play Mode

Click the ‚ñ∂ Play button in the header to test your game in real-time. All changes update instantly!

Common Issues

  • Character not showing: Make sure you assigned sprites to expressions in Character Inspector
  • Dialogue not advancing: Click anywhere or press Space/Enter
  • Variables not working: Check spelling and use curly braces: {varName}
  • Music not playing: Check browser console for errors, make sure file format is supported (MP3, OGG)
