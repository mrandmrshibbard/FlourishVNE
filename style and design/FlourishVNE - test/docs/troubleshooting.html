Common problems and how to fix them

üë• Character Issues

‚ùåCharacter not showing up in scene

‚úÖ Solution:
  1. Go to Resources ‚Üí Characters and verify the character exists
  2. Click the character ‚Üí Character Editor tab
  3. Verify you have created expressions (e.g., "neutral", "happy")
  4. Verify you have created layers and uploaded sprites
  5. Most common issue: Switch to Character Inspector tab and make sure sprites are ASSIGNED to expressions!
  6. In your scene, make sure the Show Character command is using a valid expression name

‚ùåCharacter shows but expression doesn't change

‚úÖ Solution:
  1. Open Character Inspector tab
  2. Check that EACH expression has sprites assigned to all layers
  3. Verify expression names match exactly (case-sensitive!)
  4. If using Show Character command, make sure you're selecting the correct expression from dropdown

‚ùåCharacter sprite is cut off or wrong size

‚úÖ Solution:
  1. All sprites for a character should be the same dimensions (e.g., 1000x1500px)
  2. Make sure images are PNG with transparent backgrounds
  3. Check z-index values - higher numbers appear in front
  4. Use Show Character command's position settings to adjust placement
üé¨ Scene Issues

‚ùåScene doesn't advance / stuck on dialogue

‚úÖ Solution:
  1. Click anywhere on the screen or press Space/Enter to advance
  2. If using Show Choices, you must click a choice to continue
  3. Check that your scene has more commands after the current one
  4. If scene ends, it will stay on last command (this is normal)

‚ùåVariables not working in text

‚úÖ Solution:
  1. Variables must be created in Resources ‚Üí Variables first
  2. Use curly braces: {playerName} not playerName
  3. Variable names are case-sensitive
  4. Make sure variable has a value (use Set Variable command first)

‚ùåBranching not working / wrong path

‚úÖ Solution:
  1. Every Branch Start must have a matching Branch End
  2. Check your condition logic (equal to, greater than, etc.)
  3. Verify the variable has the value you expect (use Show Text to display it: {varName})
  4. Remember: conditions are evaluated when the branch is reached, not before
üéµ Audio Issues

‚ùåMusic not playing

‚úÖ Solution:
  1. Check audio format: use MP3 or OGG (WAV files are too large)
  2. Verify file uploaded successfully in Resources ‚Üí Audio
  3. Open browser console (F12) to check for error messages
  4. For background music, make sure "Loop" is enabled
  5. Some browsers block autoplay - try clicking on the page first
  6. Check browser volume and make sure it's not muted

‚ùåMusic plays but doesn't loop

‚úÖ Solution:
  1. In Play Music command, check the "Loop" checkbox
  2. Make sure audio file is properly encoded (some formats have issues with looping)
  3. Use Play Music not Play SFX for background music
üì¶ Export Issues

‚ùåBuild Game button doesn't work

‚úÖ Solution:
  1. Open browser console (F12) to see error messages
  2. Make sure you have at least one scene created
  3. Check that all assets (images, audio) uploaded successfully
  4. Try exporting with fewer assets to isolate the problem
  5. Clear browser cache and try again

‚ùåExported game file is huge (100MB+)

‚úÖ Solution:
  1. Optimize images: use PNG for sprites, JPG for backgrounds
  2. Reduce image dimensions (1920x1080 for backgrounds, 1000x1500 for characters)
  3. Convert audio to MP3 with 128kbps bitrate
  4. Remove unused assets from Resources panel
  5. Compress videos before uploading
üé® UI Issues

‚ùåUI button not responding

‚úÖ Solution:
  1. Check that button has an action assigned in UI Element Inspector
  2. Verify the action target is valid (scene exists, screen exists, etc.)
  3. Make sure button is not covered by another UI element (check z-index)
  4. Check that button width/height are large enough to click

‚ùåTitle screen doesn't appear

‚úÖ Solution:
  1. Go to Resources ‚Üí UI Screens
  2. At the top, set your title screen in the "Title Screen" dropdown
  3. Make sure the screen has a "Start New Game" button action
  4. Verify the screen has UI elements (buttons, text, etc.)

üí° General Debugging Tips

  • Always open browser console (F12) to see error messages
  • Use Preview mode to test changes immediately
  • Start simple: test with one character, one scene, then add complexity
  • Save your project frequently (export as .vnproj file)
  • If something breaks, try refreshing the page and re-importing your project
  • Create a backup copy before making major changes

üÜò Still Having Problems?

If you've tried everything and still can't fix the issue:

  1. Create a minimal test project with just the problematic feature
  2. Check the browser console for specific error messages
  3. Try in a different browser (Chrome, Firefox, Edge)
  4. Clear browser cache and reload
  5. Report the issue with specific steps to reproduce it
